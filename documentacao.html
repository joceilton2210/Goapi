Quick Start

Comece a usar a API em menos de 5 minutos. Siga nosso guia rápido de integração.

Introdução

A GO API WhatsApp é uma solução completa para integração com o WhatsApp Business. Nossa API permite que você:

  • Envie mensagens de texto, imagens, vídeos e documentos
  • Gerencie múltiplas instâncias do WhatsApp
  • Receba webhooks em tempo real
  • Automatize respostas e fluxos de conversação
  • Integre com seus sistemas existentes
info

Base URL

https://api.goapi.com.br/v1

Autenticação

Todas as requisições à API devem incluir um token de autenticação no header. Você pode obter seu token no painel de controle.

Header de Autenticação

HTTP Header
Authorization: Bearer SEU_TOKEN_AQUI

Exemplo de Requisição

curl
curl -X POST https://api.goapi.com.br/v1/messages/send \
  -H "Authorization: Bearer SEU_TOKEN_AQUI" \
  -H "Content-Type: application/json" \
  -d '{
    "instance_id": "inst_123",
    "number": "5511999999999",
    "message": "Olá, mundo!"
  }'

Gerenciar Instâncias

POST

Criar Instância

cria uma nova sessão do WhatsApp.

Endpoint
POST /v1/instances

Body (JSON)

{ "instanceId": "minha_empresa_01" }
GET

Status da Instância

Verifica se a instância está conectada.

GET /v1/instances/:instanceId/status
GET

Obter QR Code

Retorna o QR Code em base64 para conexão.

GET /v1/instances/:instanceId/qr
DELETE

Deletar Instância

Remove a sessão e desconecta o WhatsApp.

DELETE /v1/instances/:instanceId
POST

Enviar Mensagem de Texto

Endpoint para enviar mensagens de texto simples.

Endpoint
POST /v1/messages/send

Parâmetros

Parâmetro Tipo Obrigatório Descrição
instance_id string Sim ID da instância
number string Sim Número com DDI (ex: 5511999999999)
message string Sim Texto da mensagem

Exemplo de Resposta

JSON Response
{
  "success": true,
  "message_id": "msg_abc123",
  "status": "sent",
  "timestamp": "2024-01-15T10:30:00Z"
}

Guias de Deploy e Atualização

dock_to_bottom Docker Compose (Recomendado)

Para instalar ou atualizar usando Docker Compose:

deployed_code Portainer

  1. Acesse seu painel Portainer.
  2. Vá para Containers e encontre o container da API.
  3. Clique no nome do container.
  4. No topo, clique em Recreate.
  5. Ative a opção "Pull latest image".
  6. Clique em Recreate.
  7. Isso irá baixar a nova versão e reiniciar o serviço mantendo os volumes.

PHP

example.php
<?php
function sendMessage() {
    $url = "https://api.goapi.com.br/v1/messages/send";
    $data = json_encode([
        "instance_id" => "inst_123",
        "number" => "5511999999999",
        "message" => "Olá do PHP!"
    ]);
    
    $ch = curl_init($url);
    curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
    curl_setopt($ch, CURLOPT_POSTFIELDS, $data);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_HTTPHEADER, [
        "Authorization: Bearer SEU_TOKEN",
        "Content-Type: application/json"
    ]);
    
    $result = curl_exec($ch);
    curl_close($ch);
    return json_decode($result);
}

Java

Example.java
import java.net.http.*;
import java.net.URI;

public class Example {
    public static void sendMessage() throws Exception {
        String json = "{\"instance_id\":\"inst_123\",\"number\":\"5511999999999\",\"message\":\"Olá do Java!\"}";
        
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://api.goapi.com.br/v1/messages/send"))
            .header("Authorization", "Bearer SEU_TOKEN")
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        
        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        System.out.println(response.body());
    }
}

using System.Text;

public class Example {
    public static async Task SendMessage() {

        var json = "{\"instance_id\":\"inst_123\",\"number\":\"5511999999999\",\"message\":\"Olá do C#!\"}";

    }
}

require 'json'

def send_message

  http = Net::HTTP.new(uri.host, uri.port)
  http.use_ssl = true
  
  request = Net::HTTP::Post.new(uri.path)

    message: 'Olá do Ruby!'

  response = http.request(request)

end

Go

example.go
package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func sendMessage() error {
    data := map[string]string{
        "instance_id": "inst_123",
        "number":      "5511999999999",
        "message":     "Olá do Go!",
    }
    
    jsonData, _ := json.Marshal(data)
    req, _ := http.NewRequest("POST", 
        "https://api.goapi.com.br/v1/messages/send", 
        bytes.NewBuffer(jsonData))
    
    req.Header.Set("Authorization", "Bearer SEU_TOKEN")
    req.Header.Set("Content-Type", "application/json")
    
    client := &http.Client{}
    resp, err := client.Do(req)
    return err
}

Rust

example.rs
use reqwest;
use serde_json::json;

async fn send_message() -> Result<(), Box<dyn std::error::Error>> {
    let client = reqwest::Client::new();
    
    let body = json!({
        "instance_id": "inst_123",
        "number": "5511999999999",
        "message": "Olá do Rust!"
    });
    
    let response = client
        .post("https://api.goapi.com.br/v1/messages/send")
        .header("Authorization", "Bearer SEU_TOKEN")
        .header("Content-Type", "application/json")
        .json(&body)
        .send()
        .await?;
    
    let result = response.text().await?;
    println!("{}", result);
    Ok(())
}

Códigos de Erro

A API utiliza códigos de status HTTP padrão para indicar sucesso ou falha das requisições.

Código Descrição
200 Requisição bem-sucedida
400 Requisição inválida (parâmetros incorretos)
401 Não autorizado (token inválido)
404 Recurso não encontrado
429 Muitas requisições (rate limit excedido)
500 Erro interno do servidor
